# 项目简介:用我自己的两年运动数据,做一次"较真"的科学分析

> 英文完整版见仓库根目录 `README.md`。本文是面向非专业读者的通俗简介。

## 一句话概括

这是一个**单人研究(N-of-1)**:用我自己 Garmin 手表攒下的约 **2.5 年**数据,认真分析两种截然相反的运动——耐力型的**铁人三项** vs 间歇冲刺型的**足球**(外加**力量训练**)——谁更"耗恢复"、恢复得更慢,并且全程把"怎么算、算得对不对、哪里不能下结论"都摊在明面上。

## 一、为什么做(目的)

为申请**生物统计 / 计算生物 / 数据科学**方向研究生准备一个**能拿得出手的真实作品**,证明我能把乱糟糟的真实数据变成可信结论,并且**知道分寸**——清楚说出"这个能下结论,那个不能"。

所以目的**不是**"发现一个惊天结论",而是:**用严谨、诚实的方法把一个复杂的个体系统讲清楚,并明确标出结论边界。** 严谨、诚实、可复现本身就是产出。

## 二、想回答什么(三层问题)

- **D 层(测量)**——"这些数字本身靠不靠谱?"(用胸带逐拍心跳核对手表的恢复指标 HRV)
- **A 层(核心)**——"足球是不是比铁三更耗恢复?" 分两问:同样训练量,足球当晚压得更狠吗(**成本**)?练完恢复得更慢吗(**速度**)?
- **P 层(预测,故意降级)**——"今天练了什么,能预测明天恢复好不好吗?"(当作证伪测试,预期做不到)

## 三、怎么做的

1. **拉数据**:两个 Garmin 账号(中国区 + 美国区,中国区登录很难搞、换认证方式才打通),合成 **904 天**日表 + 22 次胸带逐拍记录。
2. **先搭"防作弊"工具**:时间序列最易犯"用未来偷看过去"(数据泄漏)的错,会让结果虚高;所以第一件事就搭一套**防泄漏评估框架**,后面所有结论都过它这关。
3. **先写计划再看结果(预注册)**:像科研规范那样先把方案写死,中途每处改动记进**只增不改的修订日志(A1–A10)**。
4. **跑三层分析**:每层先在"已知答案的模拟数据"上验证方法,再上真实数据。
5. **自己审自己**:派一组 AI 审计员从 7 个角度对抗性挑错,每条再用两个视角独立复核。
6. **写成可读成果**:README + 4 张图,全部公开在 GitHub。

## 四、结果

| 层 | 问题 | 结论 |
|---|---|---|
| **D 测量** | 数字靠谱吗 | RR 解析自洽(我们的 60000/mean(RR) 与固件 avg HR 偏差约 1 bpm)——但二者出自**同一条胸带心搏流**,这是解析自洽性检查、**不是独立设备一致性**;**真正的 D 层结论是:运动中 HRV 被晃动噪声虚高约 3 倍**,足球比跑步更脏 |
| **A · 成本(H-A1)** | 足球更耗吗 | **没差异**:每单位训练量,足球和铁三对恢复的压制几乎一样 |
| **A · 速度(H-A2)** | 恢复更慢吗 | 恢复时间:铁三≈0.4 天、足球≈0.77 天、力量≈0.88 天。**足球恢复约慢 2 倍(97% 概率),但差一口气未跨过严格门槛**(足球仅 48 次,样本是瓶颈) |
| **P 预测(H-P1)** | 能预测明天吗 | **预测不了**:加上"今天练了什么"对预测几乎无提升 |

## 五、一个亮点故事

第一次跑真实数据时结果荒唐——训练好像让恢复变好了。一查发现是**时间戳错位**:手表把整夜 HRV 标在"早晨",所以今天训练的影响其实落在**下一晚**;原模型对到了"同一晚",也就是"你早上状态好才决定去练"的那一晚(典型的"状态好才练"的反向因果)。修正对齐后立刻恢复正常方向。**这种只有真实数据才会冒出来的坑,正是真分析和玩具分析的分水岭。**

此外,AI 自审还抓到一个**真 bug**:重叠存档导致训练量被重复计算。修掉后结论没变,但纠正了一个已发布数字。**"会审计自己并诚实记录"本身就是项目最值钱的地方。**

## 六、价值

不在于"爆点",而在于**做事方式**:防泄漏、预注册 + 透明修订日志、对抗性自审、133 个测试全过、代码干净可复现;**两个 null + 一个临界 + 一个 bug 修复,全部如实公开**。它展示的是"可信地做数据科学"的全套功夫。

## 七、它不能说什么(诚实边界)

- 只关于我一个人(N=1),不能推广到人群;
- 手表 HRV 是衍生摘要值,非实验室金标准,只当"个人内部趋势"用;
- 观察性数据,不是随机实验,只能说"相关"不能说"因果"(运动类型还和季节缠在一起)。
